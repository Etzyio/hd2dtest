# 项目功能清单

> 项目名称：**清风不扬 (HD-2D Wuxia JRPG)**
> 引擎：Godot 4.6 (C# / .NET 10.0)
> 总代码量：~21,000 行

---

## 一、系统架构

### 1.1 自动加载单例 (Autoload)
| 单例 | 职责 |
|---|---|
| `ConfigManager` | 用户配置（音频、图形、语言、按键），持久化到 `user://config.json` |
| `GameManager` | 游戏状态、物品背包、队友管理、UI 弹窗/取消处理 |
| `ResourcesManager` | 加载 `Resources/Static/*.json` 到类型化缓存字典（异步加载+优先级） |
| `SaveManager` | JSON 存档系统，`user://saves/`，20 槽位限制，自动存档 |
| `DialogueManager` | 对话解析、打字机效果、分支选项 |
| `VersionManager` | 版本追踪，从 `version.json` 读取，用于存档兼容性 |
| `HD2DRenderManager` | HD-2D 渲染管线控制 |

### 1.2 场景系统
- 入口场景：`main.tscn`
- 场景切换：`Main.SwitchScene("sceneName")` → `ViewRegister.json` 映射查找 `.tscn` 路径
- `GameViewRegister` 管理场景路径注册表

### 1.3 数据流
```
Resources/Static/*.json → ResourcesManager (异步优先级加载) → 静态 Dictionary 缓存
```
所有游戏数据（技能、物品、怪物、武器、装备、NPC、关卡、任务）均以 JSON 文件格式存储。`ViewRegister.json` 同步优先加载。

## 二、用户界面

### 2.1 主菜单 (Start)
- 背景渐变层 + 装饰线 + 中心面板（420×540）
- 标题 + 副标题（"清风不扬 - 武侠JRPG"）
- 按钮：开始游戏 / 继续游戏 / 设置 / 退出
- 入场动画：背景淡入（1s）→ 装饰线（0.8s）→ 按钮（0.6s）
- 版本号显示
- 自定义字体样式（秋水烧泰 Bright）

### 2.2 弹出菜单 (PopupMenu)
- ESC 打开半透明背景菜单
- 6 个功能面板导航：
  - **物品 (Items)** — 背包查看与使用
  - **地图 (Map)** — 区域地图
  - **角色 (Characters)** — 角色状态详情
  - **技能 (Skills)** — 技能管理
  - **设置 (Settings)** — 语言/音频/图形
  - **存档 (Save/Load)** — 存档管理
- 逐级返回逻辑：子菜单 → 主菜单 → 关闭

### 2.3 角色状态面板 (PopupCharacterPanel)
- **属性标签页**：职业信息（BBC 格式）、等级、战斗属性（HP/MP/ATK/DEF/SPD）、装备加成（绿色标注）
- **装备标签页**：每个槽位显示 X（卸下）/ +（装备）按钮，卡片式布局
  - 装备选择弹窗（`AcceptDialog` 内嵌可滚动列表）
- **队友标签页**：卡片形式显示队友姓名、职业、HP/MP 条
- 基础属性数值基于 `CalculateStats()` 综合计算（职业成长 × 等级 + 装备加成 + 被动加成）

### 2.4 技能面板 (PopupSkillsPanel)
- **左侧面板**：
  - 角色选择器（`OptionButton`，含 1 主角 + 5 队友）
  - 职业信息（名称、描述、熟练度）
  - JP 点数显示
  - 职业切换按钮
  - 4 个主动技能槽位 + 1 个大招槽位 + 4 个被动技能槽位
- **右侧面板**：
  - 技能池：已学习技能（未装备）+ 锁定技能（JP 成本 + Learn 按钮）
  - 跨职业技能（带 `[Cross-class]` 标记）
  - 技能详情（名称、类型、描述、MP 消耗、冷却时间、效果列表）
  - 装备到槽位按钮
- **交互**：点击槽位 → 选择技能 → 装备；X 按钮卸下技能

### 2.5 设置面板 (PopupSettingsPanel)
- 语言选择：zh_CN / en_US / ja_JP
- 其他设置项框架

### 2.6 战斗 UI (BattleUI)
- 敌我双方角色卡片（HP/MP 进度条、名称、等级）
- 回合顺序指示器
- 动作面板：攻击 / 技能 / 道具 / 防御 / 跳过
  - 技能子菜单：显示大招（★金色标记）+ 已装备主动技能（最多 4 个），按 MP 和冷却禁用
  - 道具子菜单：显示可用消耗品
- 目标选择模式（点击敌人卡片选择目标）
- 战斗消息日志
- 胜利/失败结算画面（金币、经验值、JP、掉落物品、升级信息）

### 2.7 其他 UI
- **GameUI** / **CharacterStatus** / **InventoryUI** — 游戏内 HUD
- **TalkUI** — 对话界面，打字机效果
- **Teammate** — 队友状态条
- **Minimap** — 小地图
- **SaveSlotSelector** — 存档槽位选择器

## 三、战斗系统

### 3.1 战斗管理器 (BattleManager)
- Persona 风格回合制，按速度排序
- 状态机：Setup → TurnCalculation → PlayerTurn / EnemyTurn → ActionExecution → Victory / Defeat
- 信号驱动：`TurnStarted`、`BattleEnded`
- 动作类型：攻击、技能、切换成员、防御、跳过
- 敌人 AI：简化的随机目标选择

### 3.2 伤害计算 (DamageCalculator)
- 静态工具类，计算物理/魔法伤害
- 支持伤害类型：Physical / Magic / Fire / Water / Thunder / Ice / Holy / Dark / Poison / Light

### 3.3 战斗奖励 (BattleRewards)
- 金币 = 随机（5 ~ 等级×20+5）
- 经验值 = 等级 × 10
- **JP (Job Points)** = 等级 × 2（精英 ×4，Boss ×8）
- 物品掉落：40% 概率从掉落列表抽取一件
- 升级检测：经验值 ≥ 等级×100 时升级

### 3.4 增益系统 (BuffSystem)
- Buff 生命周期管理：应用 → 每 Tick → 移除
- 堆叠/刷新规则
- `CreatureBuffExtensions` 提供便捷方法

## 四、技能系统

### 4.1 技能管理器 (SkillManager)
- 78 个技能模板（54 主动 + 6 大招 + 18 被动 + 4 主角专属）
- 技能注册：`MakeActive()` / `RegisterPassive()`
- 模板克隆创建实例：`CreateSkillInstance()` / `CreatePassiveInstance()`
- 按职业获取技能 ID 列表：`GetClassSkillIds()`

### 4.2 六职业设计

| 职业 | 特点 | HP成长 | ATK成长 | DEF成长 | SPD成长 | MP成长 |
|---|---|---|---|---|---|---|
| **Swordsman** 剑客 | 均衡近战 | 1.1 | 1.2 | 1.0 | 1.0 | 0.8 |
| **Healer** 治疗者 | 回复支援 | 0.9 | 0.6 | 0.7 | 1.0 | 1.5 |
| **Mage** 法师 | 元素毁灭 | 0.7 | 1.5 | 0.6 | 0.8 | 1.8 |
| **Tank** 坦克 | 坚不可摧 | 1.8 | 0.7 | 1.8 | 0.6 | 0.5 |
| **Assassin** 刺客 | 快速致命 | 0.8 | 1.4 | 0.5 | 1.8 | 0.6 |
| **Archer** 弓箭手 | 远程精准 | 0.9 | 1.2 | 0.7 | 1.2 | 0.8 |

### 4.3 技能效果系统
- 技能类型：Attack / Defense / Healing / Support
- `SkillDefent` 效果项：伤害系数、持续回合、伤害类型
- 技能属性：MP 消耗、冷却时间、解锁状态、**JP 消耗**

### 4.4 技能事件系统 (SkillEvents)
- 事件总线：`OnSkillCast`、`OnDamageCalculated`、`OnBuffApplied`
- 订阅/发布模式

### 4.5 技能执行器 (SkillExecutor)
- 技能流程：检查消耗 → 播放动画 → 触发效果 → 冷却管理

## 五、职业与精通系统

### 5.1 职业定义 (BattleClass)
- 数据来源：`Resources/Static/Classes.json`
- 静态注册表：按 ClassId 注册/查询
- 包含：成长系数、技能 JP 消耗、被动解锁条件、大招消耗

### 5.2 职业精通 (ClassProficiency)
- 每个角色独立追踪每个职业的精通度
- `IsUnlocked` / `ProficiencyLevel` / `LearnedSkillIds`
- 精通后可跨职业使用已学技能

### 5.3 JP 学习系统
- 战斗获得 JP，用于解锁职业技能
- 初始只自动解锁 JP 消耗为 0 的基础技能
- 大招单独 JP 购买（200~300 JP）

### 5.4 被动解锁规则
- 主动技能解锁数达到阈值时自动解锁被动
- 每个职业 4 个被动，分别在 2/4/6/8 个主动解锁时获得
- 解锁时自动装备，重新计算角色属性

### 5.5 职业切换
- 角色可切换已解锁的职业
- 技能面板的 "Switch Class" 按钮弹出职业选择对话框
- 显示所有已解锁职业及属性成长对比

## 十四、开发工具

### 14.1 工具类 (Utilities)
- `Log` — 分级日志（Info / Warning / Error / Debug / Critical）
- `SingletonHelper` — 安全单例访问模式
- 命名规范：所有日志必须使用 `Log.*` 而非 `GD.Print` 或 `Console.WriteLine`

### 14.2 构建命令
```bash
dotnet build          # Debug 模式（含 Steam）
dotnet build -c NoSteam  # 无 Steam
dotnet build -c Release  # 发布模式
bash build_version.sh    # 从 git 元数据生成 version.json
```
